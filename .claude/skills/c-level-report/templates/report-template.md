# C-Level Report Template — AI Trading Agent Platform

> This file defines the structural template for every executive report generated by the `/c-level-report` skill.
> Claude reads this file during generation and fills each `{PLACEHOLDER}` with data gathered from
> `development/context.md`, git log, test inventories, code reviews, and agent memory files.

---

## Frontmatter Template

```yaml
---
type: c-level-report
date: {YYYY-MM-DD}
scope: {full|progress|quality|risk|agents|roadmap}
generated-by: c-level-report-skill
platform: AI Trading Agent
version: "1.0"
tags:
  - executive
  - status-report
---
```

## Visual Elements Reference

### Progress Bars

Use block characters `█` (filled) and `░` (empty) over 10 positions. Append percentage label.

| Completion | Progress Bar                  | Label  |
|------------|-------------------------------|--------|
| 0%         | `░░░░░░░░░░`                  | 0%     |
| 20%        | `██░░░░░░░░`                  | 20%    |
| 40%        | `████░░░░░░`                  | 40%    |
| 60%        | `██████░░░░`                  | 60%    |
| 80%        | `████████░░`                  | 80%    |
| 100%       | `██████████`                  | 100%   |

Example usage in a table cell: `████████░░ 80%`

### Status Icons

| Icon | Meaning              | When to Use                                      |
|------|----------------------|--------------------------------------------------|
| ✅   | Complete / Healthy   | Task done, service up, test passing, no issues   |
| ⚠️   | Warning / At Risk    | Partial completion, known issues, deferred items |
| ❌   | Blocked / Failed     | Task blocked, service down, test failing         |
| 🔄   | In Progress          | Actively being worked on, partially complete     |

### Trend Arrows

| Arrow | Meaning     | Context Example                                     |
|-------|-------------|-----------------------------------------------------|
| ↑     | Improving   | Test count rising, error count falling, perf better |
| ↓     | Degrading   | Error count rising, coverage dropping               |
| →     | Stable      | No meaningful change since last report              |

### Table Alignment Conventions

- Left-align all text columns (names, descriptions, notes)
- Right-align numeric columns (counts, percentages) when practical
- Center-align status and icon columns
- Use `—` for cells where data is not applicable
- Never leave a cell blank — use `{VALUE}` if data was not found, or `—` if truly N/A
