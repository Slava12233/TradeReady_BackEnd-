# Code Reviews

<!-- last-updated: 2026-03-18 -->

This folder contains code review reports generated by the `code-reviewer` agent after each review.

## Purpose

Provides a persistent, browsable history of code reviews so the team can track recurring issues, monitor code quality trends, and reference past decisions.

## File Naming Convention

Reports follow the pattern:
```
review_YYYY-MM-DD_HH-MM_{scope}.md
```

- `scope` is a short descriptor of what was reviewed (e.g., `strategy-routes`, `battle-service`, `auth-middleware`)
- Example: `review_2026-03-18_14-30_strategy-routes.md`

## Report Structure

Each report contains:
- **Date and reviewer** (always `code-reviewer agent`)
- **Files reviewed** with paths
- **CLAUDE.md files consulted**
- **Verdict**: `PASS`, `PASS WITH WARNINGS`, or `NEEDS FIXES`
- **Critical issues** (must fix) with file, line, rule violated, and fix
- **Warnings** (should fix)
- **Suggestions** (optional improvements)
- **Passed checks** summary

## Usage

- Reports are append-only — never edit or delete past reviews
- The code-reviewer agent creates a new report after every review
- Use these reports to identify recurring patterns or frequently violated rules
