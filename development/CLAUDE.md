## Purpose

This directory contains **all internal planning documents, task trackers, research reports, and development context**. These are reference-only archives — the source of truth is always the code itself. Use this folder to understand the history, motivation, and progress of development phases.

> **Start here:** Read `context.md` first — it is the rolling summary of all development activity maintained by the `context-manager` agent.

## Obsidian Vault

This directory doubles as an **Obsidian vault** for human knowledge management. The `.obsidian/` config folder makes it recognizable as a vault. See `VAULT-SETUP.md` for first-time setup instructions.

**Key conventions:**
- All markdown files have YAML frontmatter with `type`, `tags`, and other metadata fields
- `[[wikilinks]]` are used for cross-references within `development/` files only — never in CLAUDE.md files or source code
- Dataview plugin queries power dashboards in `_dashboards/`
- Daily notes in `daily/` provide a shared human+agent development log

**Vault-specific directories (not for agent navigation):**

| Directory | Description |
|-----------|-------------|
| `.obsidian/` | Obsidian config (plugins: Dataview, Templater, Git) |
| `_templates/` | Obsidian Templater templates (code-review, task, daily-note, plan, research-report) |
| `_dashboards/` | Dataview dashboard notes (project-health, agent-activity) |
| `_moc/` | Map of Content index notes (Home, task-boards, code-reviews, plans, research, agents, daily-log, git-workflow, wikilink-conventions) |
| `_attachments/` | Image/file attachments for vault notes |
| `daily/` | Daily development log notes (YYYY-MM-DD.md format) |

## How to Use This Folder

- **Starting a new conversation?** → Read `context.md` for full current state
- **Understanding a past decision?** → Check `context.md` "Key Design Decisions" section
- **Planning a new feature?** → Review existing plans for patterns and conventions
- **Checking what's done?** → `context.md` "What's Built" table has the definitive status
- **Looking for code review history?** → Check `code-reviews/` subdirectory

## Patterns

- All task files use checkbox format: `- [x] Done` / `- [ ] Pending`
- Planning docs follow: Problem → Research → Design → Implementation Plan → Tasks
- `context.md` is the only file actively maintained (by `context-manager` agent)
- Research report files (`*-report.md`) are reference material from codebase analysis
- All other files are frozen after their phase completes
