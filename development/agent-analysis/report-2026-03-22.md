### Top Recurring Issue Types

1. **Bare `except:` / overly broad `except Exception`** -- Found in 4 of 10 reviews. The most persistent anti-pattern. Agent code uses `except Exception` 5+ times where `httpx.HTTPStatusError` would suffice.
2. **Financial float precision** -- Found in 3 reviews. `float(Decimal)` passed to Redis, floats in test factories. CRITICAL-1 in permissions review.
3. **Missing React memoization** -- Found in 2 frontend reviews. Zustand selectors re-created every render, header components subscribing to too many stores.
4. **Stale documentation claims** -- Found in 2 reviews. Changes documented as done but code not actually updated (CSS extraction, rAF switch).
5. **Unsafe deserialization (pickle)** -- Found in 2 security reviews. `PPO.load()` and `joblib.load()` without SHA-256 verification (deferred HIGH).

### False Positive Patterns (confirmed by code-reviewer memory)

These are NOT issues despite looking like them:

- `# noqa: PLC0415` on lazy imports in `src/dependencies.py` -- intentional circular import avoidance
- `# type: ignore[prop-decorator]` on computed_field -- known pydantic v2 limitation
- `asyncio_mode = "auto"` means no `@pytest.mark.asyncio` needed -- absence is correct
- `# noqa: BLE001` on `except Exception` in LLM steps -- pydantic-ai raises varied types

No new false positives detected. The code-reviewer memory already captures all known false positive patterns.

## Agent Prompt Improvements

### 1. backend-developer: Add bulk-rename guidance

The 34-task sprint showed that structlog event name migration via individual `Edit` calls is extremely verbose (24 edits to one file). Add to backend-developer prompt:

> **Bulk string replacement:** For project-wide renames (e.g., log event name migration), write a Python script to perform all replacements in one pass rather than issuing individual Edit calls. Place scripts in `scripts/` with a descriptive name.

### 2. test-runner: Add coverage gap detection for sprints

During the 34-task sprint, only 6 events touched `tests/` despite 34 tasks being completed. Add:

> **Sprint coverage check:** After a multi-task sprint, run `pytest --co -q` to count discovered tests and compare with the pre-sprint count. Flag if new code files lack corresponding test files.

### 3. code-reviewer: Add documentation claim verification

Two reviews found that documented changes were not actually applied (CSS extraction, rAF switch). Add:

> **Documentation claim verification:** When `context.md` or task descriptions claim a change was made, verify the actual file content matches the claim. Flag discrepancies as CRITICAL.

### 4. context-manager: Add cross-agent data staleness check

Four agents had stale migration head or test count values. Add:

> **Cross-agent sync check:** After updating `context.md`, verify that key facts (migration head, test counts, CLAUDE.md count) are consistent across all agent memory files. Flag inconsistencies for manual correction.

### 5. security-auditor: Expand audit coverage tracking

The "Areas NOT Yet Audited" section is valuable. Add a periodic reminder:

> **Audit coverage goal:** Track percentage of codebase audited. Current: ~35% (agent/permissions, agent/strategies, phase 3 logging). Priority unaudited areas: `src/api/routes/`, `src/accounts/`, `src/order_engine/`, `Frontend/`.

## Sprint Observations (2026-03-21 Agent Logging System)

### What Went Well

- **Single-session throughput:** 798 tool calls in 2.5 hours = 5.3 calls/minute sustained
- **Edit dominance:** 76% Edit calls indicates the agents were modifying existing code (correct behavior for a refactoring sprint)
- **Memory seeding:** All 16 agent memories were populated in the same session -- no gaps
- **Zero regressions reported:** No test failures attributed to the logging changes

### What Could Improve

1. **Bulk operations:** The structlog migration (Tasks 05-06) generated the most edits. A script would have been 10x more efficient.
2. **Test coverage ratio:** 6 test-related events out of 798 (0.75%) is low for a sprint that added 66 tests.
3. **Cross-project leakage:** 30 events logged against `claude-code-starter-kit/` -- the logging hook should filter by project root.
4. **Path normalization:** Same file appears with both `\` and `/` separators (e.g., `agent/tools/rest_tools.py` vs `agent\tools\rest_tools.py`). The logging script should normalize paths.
